# 3D合戦ページ スクショ検証手順

## 目的

各合戦ページが、要件定義に書いた「部隊数・時刻・移動方向・見えるべきもの」と一致しているかをスクリーンショットで確認する。

## 検証モード

各合戦ページは次のクエリを利用できる。

- `verify=1`: 再生を止め、検証用の俯瞰カメラにする
- `t=<時刻>`: 指定時刻で停止する
- `weather=0`: 霧・雨を極力抑えて動線確認を優先する

例:

```text
http://127.0.0.1:8766/battles/sekigahara/?verify=1&t=12&weather=0
http://127.0.0.1:8766/battles/okehazama/?verify=1&t=13.4&weather=0
http://127.0.0.1:8766/battles/kawanakajima/?verify=1&t=8&weather=0
http://127.0.0.1:8766/battles/nagashino/?verify=1&t=10&weather=0
http://127.0.0.1:8766/battles/yamazaki/?verify=1&t=17&weather=0
http://127.0.0.1:8766/battles/mikatagahara/?verify=1&t=16&weather=0
```

## スクショ観点

- 部隊ラベルが読める
- 地形名が読める
- 該当時刻の字幕が出る
- 要件定義の方向へ部隊が移動している
- 霧・雨が演出として見えるが、部隊と動線を完全に隠さない
