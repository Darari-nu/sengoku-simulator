# 戦国合戦 3D絵巻 実装計画 v1.0

作成日: 2026-06-28

この計画は、`REQUIREMENTS.md` を正の要件定義書として、誤って先に作られた旧プロトタイプを整理しながら「戦国合戦 3D絵巻」を静的GitHub Pagesサイトとして育てるための実装方針をまとめる。

---

## 前提

- 正とする仕様: `REQUIREMENTS.md`
- 合戦ページの核: `battles/sekigahara/index.html`
- 旧プロトタイプ: ルートの `simulator.js` / `battle-data.js` / `style.css` と旧 `sekigahara/` は削除済み
- 公開方式: GitHub Pages の静的HTML/CSS/JS
- ビルドツール: Phase 1では導入しない
- 3Dライブラリ: Three.js
- ブランド表現: 特定放送局名は使わず、「大河ドラマ風」「重厚な歴史番組風」と表現する

## Phase 2: 関ヶ原を基準作品として磨く

目的: `battles/sekigahara/index.html` を今後の合戦ページの品質基準にする。

対応内容:

- OGP画像用の関ヶ原スクリーンショットを作る
- メタ情報と史実注記を整える
- モバイルで操作UIが重ならないか確認する
- Playwrightまたは手動スクリーンショットで表示確認する
- `battles/sekigahara/index.html` の構造を「次の合戦で真似する単位」に分解してメモする

品質基準:

- 初回表示で3Dシーンが空白にならない
- 時刻スライダーで部隊配置・カメラ・字幕が変化する
- スマホでも最低限、再生・停止・視点切替が操作できる
- 史実の厳密再現ではなく、理解しやすい可視化であることを明記する

### 桶狭間の戦い

見せたい体験:

- 今川本隊の大軍が進む
- 織田勢が豪雨・地形を利用して奇襲する
- 狭い谷筋や丘陵で視界が悪くなる

必要データ:

- 織田信長、今川義元、松平元康、今川本隊、織田奇襲隊

### 長篠の戦い

- 武田軍の騎馬突撃が馬防柵にぶつかる
- 織田・徳川連合軍の鉄砲隊が段階的に射撃する
- 雨後の地形、柵、川、丘陵が突撃の失敗を説明する

必要データ:

- 織田信長、徳川家康、武田勝頼、酒井忠次、馬場信春、山県昌景
- 時間軸: 布陣、武田軍前進、鉄砲射撃、重臣討死、武田軍崩壊

目的: 合戦ごとに単一HTMLを増やし続ける限界が見えたら、共通エンジン化する。

共通化候補:

- `scripts/battle-simulator.js`
- `styles/battle.css`
- `data/battles/<battle-id>.js`
- `data/battles-index.js`

共通化する条件:

- 3本以上の合戦ページが公開済み
- 同じThree.js初期化処理を複数ページで修正する必要が出た
- UIやカメラ制御の差分が小さく、データだけで表現できる見通しが立った

Phase 1では共通化しない。まずは完成品を壊さず増やす。

## プログラム構成方針

### Phase 1の構成

トップページ:

- `index.html`: トップのHTML
- `styles/index.css`: トップ用の見た目
- `data/battles-index.js`: 合戦カード用データ

関ヶ原ページ:

- `battles/sekigahara/index.html`: Three.js、CSS、合戦データを含む単一HTML

追加合戦ページ:

- `battles/okehazama/index.html`
- `battles/kawanakajima/index.html`

### 合戦ページの基本構造

`battles/sekigahara/index.html` を参考に、各合戦ページは当面この構造で作る。

- DOM: タイトル、時刻表示、視点チップ、字幕、タイムライン
- Three.js初期化: `WebGLRenderer`、`Scene`、`PerspectiveCamera`
- 地形生成: 合戦ごとの山・川・街道を簡易メッシュで表現
- 部隊データ: 武将名、陣営、兵力、説明、時間別の移動パス
- 演出データ: 字幕、矢印、霧、煙、光
- ループ: 時刻を進め、部隊・カメラ・UIを更新

Three.jsは、公式ドキュメントで確認した基本形に合わせて、`WebGLRenderer`、`PerspectiveCamera`、リサイズ時の `camera.aspect` 更新、`requestAnimationFrame` または `renderer.setAnimationLoop` による描画ループを使う。

## 実装順

1. `.nojekyll` を追加する
2. `battles/sekigahara/index.html` を関ヶ原完成版として配置する
3. `data/battles-index.js` に3合戦のカード情報を書く
4. `styles/index.css` を作る
5. ルート `index.html` をトップページに作り替える
6. 桶狭間・川中島の3D合戦ページを作る
7. `404.html` を作る
8. ローカルサーバーで表示確認する
9. 問題なければ旧プロトタイプを削除する
10. コミットしてGitHub Pagesで確認する
